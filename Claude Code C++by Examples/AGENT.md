# AGENT.md

本檔是此專案的 Codex / Agent 工作規範。所有後續對話、掃描、修改與新增範例，都必須遵守以下規則。

## 回答語言

1. 永遠使用繁體中文回答使用者問題，不論使用者用中文或英文提問。

## C++ 教學範例原則

2. 每個主題都要盡量對照相對應的 LeetCode 題目，但要選擇簡單、實用、能學以致用的題型，不加入為了考演算法而過度刁鑽的題目。
   - LeetCode 題目要加入現有 C++ 程式碼的註釋或範例區塊中。
   - 範例必須有完整、詳細、能直接閱讀學習的註釋。
   - 若該主題不適合硬套 LeetCode，例如 filesystem、chrono、exception、cast 等日常 API 主題，可以改放每天工作會遇到的實用案例，但仍要說明可對照的題型或使用場景。

3. 每個程式只聚焦一個主題。
   - 每個主題都要有詳細完整的課題介紹與觀念解釋。
   - 課題介紹與觀念解釋要放在程式碼註釋中。
   - 註釋可以遠多於原本程式碼，目標是讓使用者直接看 source code 就能像看課堂筆記一樣學習。

4. 所有程式範例都要簡單直接。
   - 不要刻意寫艱澀的 C++。
   - 不要為了展示技巧而拉長程式碼。
   - 優先使用清楚、短小、可編譯、容易修改的範例。

5. 除了 LeetCode 範例，也可以加入每天工作可能會用到的現成實用範例。
   - 工作案例要簡潔、實際、容易套用。
   - 一樣必須有完整詳細的註釋說明「為什麼這樣寫」與「何時該用」。

6. 每個 source code 檔案都要有課堂一般的筆記。
   - 筆記要記錄相關知識、觀念、陷阱、時間複雜度、常見用法、實務建議。
   - 目標是讓使用者直接閱讀程式碼就能學習，不需要另外查大量外部教材。
   - 不要只是新增每個 source code 的標題、檔名導讀或閱讀方式；新增內容必須是實際的 C++ 知識、語法、API 用法、範例解說、常見錯誤或實務注意事項。
   - 註釋應盡量靠近相關程式碼、函式、include、資料結構或範例段落，讓讀者看到程式碼時能同步理解語法與用法。
   - 註釋深度要假設使用者手邊沒有教科書；不能只寫三行摘要。要像課本旁註一樣說明：
     - 這個 C++ 主題/函式解決什麼問題。
     - 什麼情況適合使用，什麼情況不適合。
     - 正確使用步驟與前置條件。
     - 容易犯的錯、undefined behavior、iterator/reference 失效、生命週期問題。
     - 時間/空間複雜度與實務成本。
     - 可連結的 LeetCode 題型或每天工作會遇到的案例。
   - 即使檔案已經有 Claude Code 產生的註釋，也要用 Codex 角度補上缺漏的觀念、使用判斷、陷阱與實務經驗。

7. 所有 C / C++ 標準函式庫、語言特性與行為說明，優先參考：
   - https://en.cppreference.com/cpp
   - https://cplusplus.com/reference/
   - 若兩者資訊不同，以 cppreference 與 C++ 標準語意為優先；cplusplus.com 可作為較易讀的輔助速查。
   - 註釋中需要提到 iterator invalidation、time complexity、undefined behavior、implementation-defined behavior、exception guarantee 等重點時，要盡量查清楚再寫。

8. 所有 `summary.cpp` 檔案都要像教科書一樣有課堂筆記式註釋。
   - `summary.cpp` 只整理主題、觀念、API 對照、常見陷阱與實務建議。
   - `summary.cpp` 不加入任何 LeetCode 題目或 LeetCode 範例。
   - 若需要 LeetCode 題型，放在一般主題 `.cpp` 檔案，不放在 `summary.cpp`。
   - `summary.cpp` 不要只加章節標題；必須補充該章節真正需要知道的 C++ 語法、標準函式庫用法、限制、陷阱與實務判斷。
   - `summary.cpp` 的補充應像章末總整理：列出本章 API 的選擇準則、共通前置條件、錯誤模型、複雜度、失效規則與常見誤用；不要只寫「本章要注意什麼」的短句。

## 掃描與更新準則

- 掃描 source code 時，主要檢查 `.cpp` 檔案。
- 無副檔名且看起來像編譯產物的檔案，不視為 source code，不主動修改。
- 新增註釋時不要改變原本程式邏輯。
- 若只是在補課堂筆記、LeetCode 對照或實務案例說明，優先用註釋區塊完成，避免破壞既有可編譯範例。
- 若要批次補註釋，不能只套用泛用模板標題；必須依目錄與檔案主題補上有內容的 C++ 知識點。
- 批次補註釋時，所有 `.cpp` 都要納入，不只 `summary.cpp` 或缺少 LeetCode 的檔案。
