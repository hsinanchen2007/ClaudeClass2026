# C++ OOP 學習專案規範

本專案為 C++ 物件導向程式設計學習筆記，從初學者到進階程式設計師循序漸進。

## 通用規則

1. **語言**：永遠用繁體中文回答使用者的問題，不論使用者是用中文或英文提問。

2. **Leetcode 題目整合**：每個主題都要參考相對應的 Leetcode 題目，但只挑選**簡單實用**的題目，不要那種為了考算法而刁鑽不實用的題目。把題目加入現有的 C++ 程式碼範例中，這樣可以學以致用，並且要加上完整詳細的註釋。

3. **單一主題、詳盡註釋**：每個程式檔案只能有一個主題。每個主題都要有詳細完整的「課題介紹」與「概念解釋」，內容要完整詳細，放在程式碼裡面的註釋中。**註釋的篇幅遠多過實際程式碼也沒關係**，重點是要讓人讀程式碼就能像讀課堂筆記一樣學會該主題。

4. **範例要簡單直接**：所有程式範例都要簡單直接，**不要寫艱澀難懂的 C++ 程式碼或很長的範例**。重點是讓初學者也能讀懂。

5. **實用工作範例**：除了 Leetcode 範例之外，若有現成實用、每天工作可能會用到的應用情境，也可以加入程式碼範例。記得保持**簡潔簡單**，並附上完整詳細的解說註釋。

6. **官方參考來源**：所有 C/C++ 語法與標準函式庫的參考，請以以下兩個網站為準：
   - https://en.cppreference.com/cpp
   - https://cplusplus.com/reference/

## 檔案命名規則

- 檔名格式：`<編號>_<主題名稱>.cpp`，例如 `1_HelloClass.cpp`、`2_Constructor.cpp`、`3_Encapsulation.cpp`。
- 編號從 1 開始遞增，依照學習順序由淺入深排列。
- 每個 `.cpp` 檔案都必須是**自包含**（self-contained）的：包含自己的 `main()` 函式，可以獨立編譯執行。

## 編譯驗證

- 每建立一個新的範例檔案，**都必須使用 g++ 編譯一次**確認程式正確無誤。
- 建議編譯指令：`g++ -std=c++17 -Wall -Wextra <檔名>.cpp -o <輸出名>`
- 編譯成功後再執行一次，確認輸出符合預期。

## 註釋風格

- 檔頭區塊：用大段註釋介紹本檔案的主題、學習目標、相關概念、Leetcode 題目編號（若有）。
- 程式碼內：在每個關鍵語法、關鍵字、語意上加上「為什麼這樣寫」的繁體中文註釋。
- 範例輸出：在 `main()` 後或檔尾用註釋示範執行結果，方便對照閱讀。
