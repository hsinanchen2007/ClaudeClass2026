# CLAUDE.md

此檔案為 Claude Code 在此工作目錄下協助使用者時須遵守的指引。

## 互動與教學原則

1. **語言**：永遠使用「繁體中文」回答我的問題，不論我用中文或英文提問。

2. **Leetcode 題目對應**：每個主題都要參考對應的 Leetcode 題目，並把題目實作整合到 C++ 程式碼裡，以達到「學以致用」的目的。
   - 只挑「簡單實用」的題目，避免那種為了考算法而刁鑽、實際工作上用不到的題目。
   - 程式碼必須附上完整詳細的註釋（解題思路、複雜度、邊界條件等）。

3. **單一主題單一檔案**：每個程式檔案只能聚焦在一個主題上。
   - 該主題必須附上「完整詳細的課題介紹與觀念解釋」，寫在程式碼的註釋裡。
   - 註釋的篇幅遠多過程式碼本身也沒關係，重點是把觀念講清楚。

4. **簡單直接的範例**：所有程式範例都要簡單直接。
   - 不寫艱澀的 C++ 寫法（避免過度模板、過度抽象、過度 metaprogramming）。
   - 不寫又臭又長的程式碼，能短就短。

5. **實用範例優先**：除了 Leetcode 範例，若有「每天工作上可能會用到」的現成實用範例，也加入程式碼。
   - 同樣保持簡潔簡單。
   - 同樣要附上完整詳細的解說註釋。

6. **課堂筆記式註釋**：每個 source code 檔案都要寫得像課堂筆記一樣。
   - 紀錄所有相關的知識點與解釋，越詳細越好。
   - 目標：我可以直接打開程式碼閱讀，就能完整學到該主題的知識。

7. **C/C++ 參考來源**：所有 C/C++ 語法、標準函式庫、行為的查證，請以下列兩個官方等級的參考為準：
   - https://en.cppreference.com/cpp
   - https://cplusplus.com/reference/

   遇到不確定的細節（例如某個 STL 容器的 complexity、某個函式的 behavior、某個語言特性的版本支援），優先查上述兩個網站再下筆。
